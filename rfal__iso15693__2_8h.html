Code an ISO15693 compatible frame.

This function takes length bytes from buffer, perform proper encoding and sends out the frame to the ST25R391x.

Parameters
[in]buffer: data to send, modified to adapt flags.
[in]length: number of bytes to send.
[in]sendCrc: If set to true, CRC is appended to the frame
[in]sendFlagsIf set to true, flag field is sent according to ISO15693.
[in]picopassMode: If set to true, the coding will be according to Picopass
[out]subbit_total_length: Return the complete bytes which need to be send for the current coding
[in,out]offset: Set to 0 for first transfer, function will update it to point to next byte to be coded
[out]outbuf: buffer where the function will store the coded subbit stream
[out]outBufSize: the size of the output buffer
[out]actOutBufSize: the amount of data stored into the buffer at this call
Returns
ERR_IO : Error during communication.
ERR_AGAIN : Data was not coded all the way. Call function again with a new/emptied buffer
ERR_NO_MEM : In case outBuf is not big enough. Needs to have at least 5 bytes for 1of4 coding and 65 bytes for 1of256 coding
ERR_NONE : No error.

Receive an ISO15693 compatible frame.

This function receives an ISO15693 frame from the ST25R391x, decodes the frame and writes the raw data to buffer.

Note
Buffer needs to be big enough to hold CRC also (+2 bytes)
Parameters
[in]inBuf: buffer with the hamming coded stream to be decoded
[in]inBufLen: number of bytes to decode (=length of buffer).
[out]outBuf: buffer where received data shall be written to.
[in]outBufLen: Length of output buffer, should be approx twice the size of inBuf
[out]outBufPos: The number of decoded bytes. Could be used in extended implementation to allow multiple calls
[out]bitsBeforeCol: in case of ERR_COLLISION this value holds the number of bits in the current byte where the collision happened.
[in]ignoreBits: number of bits in the beginning where collisions will be ignored
[in]picopassMode: if set to true, the decoding will be according to Picopass
Returns
ERR_COLLISION : collision occured, data uncorrect
ERR_CRC : CRC error, data uncorrect
ERR_TIMEOUT : timeout waiting for data.
ERR_NONE : No error.
