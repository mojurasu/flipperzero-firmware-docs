◆ rfalT4TPollerComposeCAPDU()

ReturnCode rfalT4TPollerComposeCAPDU ( const rfalT4tCApduParam *  apduParam)

T4T Compose APDU.

This method computes a C-APDU according to NFC Forum T4T and ISO7816-4.

If C-APDU contains data to be sent, it must be placed inside the buffer rfalT4tTxRxApduParam.txRx.cApduBuf.apdu and signaled by Lc

To transceive the formed APDU the ISO-DEP layer shall be used

See also
rfalIsoDepStartApduTransceive()
rfalIsoDepGetApduTransceiveStatus()
rfalT4TPollerParseRAPDU()
Warning
The ISO-DEP module is used to perform the tranceive. Usually activation has been done via ISO-DEP activatiavtion. If not please call rfalIsoDepInitialize() before.
Parameters
[in,out]apduParam: APDU parameters apduParam.cApduLen will contain the APDU length
Returns
ERR_PARAM : Invalid parameter
ERR_PROTO : Protocol error
ERR_NONE : No error

◆ rfalT4TPollerParseRAPDU()

ReturnCode rfalT4TPollerParseRAPDU ( rfalT4tRApduParam *  apduParam)

T4T Parse R-APDU.

This method parses a R-APDU according to NFC Forum T4T and ISO7816-4. It will extract the data length and check if the Satus word is expected.

Parameters
[in,out]apduParam: APDU parameters apduParam.rApduBodyLen will contain the data length apduParam.statusWord will contain the SW1 and SW2
Returns
ERR_REQUEST : Status word (SW1 SW2) different from 9000
ERR_PARAM : Invalid parameter
ERR_PROTO : Protocol error
ERR_NONE : No error
