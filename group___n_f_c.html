uint32_t  tx_flag 
)

RFAL NFC Start Data Exchange.

After a device has been activated, it starts a data exchange. It handles automatically which interface/protocol to be used and acts accordingly.

In Listen mode the first frame/data shall be sent by the Reader/Initiator therefore this method must be called first with txDataLen set to zero to retrieve the rxData and rcvLen locations.

Parameters
[in]txData: data to be transmitted
[in]txDataLen: size of the data to be transmitted
[out]rxData: location of the received data after operation is completed
[out]rvdLen: location of thelength of the received data
[in]fwt: FWT to be used in case of RF interface. If ISO-DEP or NFC-DEP interface is used, this will be ignored
Returns
ERR_WRONG_STATE : Incorrect state for this operation
ERR_PARAM : Invalid parameters
ERR_NONE : No error

◆ rfalNfcDiscover()

ReturnCode rfalNfcDiscover ( const rfalNfcDiscoverParam *  disParams)

RFAL NFC Discovery.

It set the device in Discovery state. In discovery it will Poll and/or Listen for the technologies configured, and perform Wake-up mode if configured to do so.

The device list passed on disParams must not be empty. The number of devices on the list is indicated by the devLimit and shall be at >= 1.

Parameters
[in]disParams: discovery configuration parameters
Returns
ERR_WRONG_STATE : Incorrect state for this operation
ERR_PARAM : Invalid parameters
ERR_NONE : No error
